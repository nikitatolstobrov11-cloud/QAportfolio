# Проект: Demoblaze.com

## Описание проекта
Учебный веб-сайт для тестирования интернет-магазина электроники.  
Цель: проверить функциональность магазина, корректность работы корзины, оформления заказа и пользовательского интерфейса.

## Баги

### BUG-01 - Заказ оформляется с пустыми обязательными полями
**Severity:** High  
**Шаги воспроизведения:**  
1. Открыть [https://www.demoblaze.com](https://www.demoblaze.com)  
2. Добавить товар в корзину  
3. Перейти в Cart -> Place Order  
4. Оставить пустыми поле адреса и страны 
5. Нажать Purchase  

**Ожидаемый результат:** Появляется валидация, заказ не оформляется, пользователь получает сообщение об ошибке.  
**Фактический результат:** Заказ успешно оформляется, появляется окно с подтверждением покупки.  
**Скриншот:**  
![BUG-01](QAportfolio/баг1.png)

### BUG-02 - Поле “Name” принимает числовые значения
**Severity:** Medium  
**Шаги:**  
1. Добавить товар в корзину  
2. Cart -> Place Order  
3. В поле Name ввести: 123456  
4. Заполнить остальные поля  
5. Нажать Purchase  

**Ожидаемый результат:** Поле должно принимать только буквенные значения или выдавать ошибку.  
**Фактический результат:** Числовое имя принимается, заказ оформляется.  
**Скриншот:**  
![BUG-02](ссылка_на_скриншот_в_репозитории)

### BUG-04 - Корзина не очищается после оформления заказа
**Severity:** Medium  
**Шаги:**

2. Оформить заказ  
3. Закрыть окно подтверждения  
4. Перейти снова в Cart  

**Ожидаемый результат:** Корзина пустая после успешной покупки.  
**Фактический результат:** Товар остаётся в корзине.  
**Скриншот:**  
![BUG-04](ссылка_на_скриншот_в_репозитории)

### BUG-05 - Можно добавить один и тот же товар бесконечное число раз без ограничения
**Severity:** Low / Medium  
**Шаги:**  
1. Открыть страницу товара  
2. Нажать Add to cart 5–6 раз подряд  
3. Перейти в Cart  

**Ожидаемый результат:** Либо ограничение, либо увеличение количества одного товара.  
**Фактический результат:** Товар добавляется отдельными строками без контроля.

## Итоги проекта
- Проведено функциональное тестирование всех категорий товаров  
- Созданы тест-кейсы и чек-листы для основных пользовательских сценариев  
- Выявлены баги разной серьёзности, подготовлены баг-репорты  
- Проверена корректность корзины и оформления заказа  
- Инструменты: Jira (для баг-репортов), TestRail (для тест-кейсов), Chrome DevTools, Postman
